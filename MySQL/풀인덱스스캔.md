# 풀인덱스스캔이란?

풀인덱스스캔이란 인덱스를 사용하기는 하지만 인덱스의 처음부터 끝까지 모두 읽는것을 말한다.
풀테이블스캔보다는 빠르지만 인덱스 레인지 스캔보다는 훨씬느리다

# 실행계획에는 어떻게 표시될까? 
* type: index
* (주로) Extra: Using index (커버링 인덱스일 경우)

# 예시
```sql
SELECT COUNT(*) FROM EMP
```
이런 쿼리는 풀 테이블 스캔이 아닌 풀 인덱스 스캔을 사용한다. 보통의 인덱스는 두세개의 컬럼으로만 이루어져 있어 훨씬 용량이 작아 효율적이기 때문이다.
