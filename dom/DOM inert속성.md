# inert 속성 가이드

## 1. 개요

`inert`는 특정 HTML 요소와 그 하위 노드 전체를 **비활성 상태**로 만드는 글로벌 속성이다. 이 속성이 부여된 영역은 사용자의 모든 상호작용 시도에서 무시된다.

---

## 2. 주요 기능

* **포커스 차단**: 해당 요소와 하위 요소는 Tab 키를 이용한 포커스 이동 순서에서 완전히 제외된다.
* **사용자 입력 무시**: 클릭, 터치, 드래그 등 모든 포인터 이벤트가 발생하지 않는다.
* **선택 및 검색 제한**: 영역 내 텍스트 선택이 불가능하며, 브라우저의 '찾기(Ctrl+F)' 기능에서도 검색되지 않는다.
* **보조 공학 도구 은폐**: 스크린 리더 등 보조 기술이 해당 영역을 인식하지 않도록 하여 접근성 트리에서 제거된 것과 같은 효과를 준다.

---

## 3. 코드 적용 예시

### 일반 HTML

```html
<div inert>
  <button onclick="alert('Hello')">클릭 불가</button>
  <input type="text" placeholder="입력 불가">
</div>

```

### React (v18 이하)

React 18 이하 버전에서는 `inert`가 표준 속성으로 등록되기 전이므로 아래와 같이 우회하여 적용해야 한다.

```tsx
<div {...({ inert: isDisabled ? "" : undefined } as any)}>
  <MyComponent />
</div>

```

---

## 4. 기존 방식과의 비교

| 구분 | `pointer-events: none` | `inert` 속성 |
| --- | --- | --- |
| **마우스 클릭** | 차단 가능 | 차단 가능 |
| **키보드 포커스** | **허용됨** | **차단됨** |
| **텍스트 선택** | 경우에 따라 허용 | **차단됨** |
| **스크린 리더** | **내용을 읽어줌** | **완전히 무시함** |

---

## 5. 지원 현황 및 결론

* **지원 범위**: 2026년 현재 Chrome, Edge, Firefox, Safari 등 모든 모던 브라우저에서 공식 지원한다.
* **하위 호환성**: 레거시 브라우저 대응이 필요한 경우 `WICG-inert` 폴리필을 활용할 수 있다.
* **결론**: 모달 팝업 뒤쪽의 콘텐츠를 차단하거나, 로딩 중 화면 전체의 액션을 막아야 할 때 가장 안전하고 표준화된 해결책이다.
