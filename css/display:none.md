# display: none
웹페이지에서 요소를 완전히 제거한것처럼 숨기는 속성, 핵심은 눈에만 안보이는것이 아니라 자리를 차지하던 공간까지 없애버린다. 

1. 시각적 제거: 화면에서 아예 사라진다.
2. 공간 제거 (Layout): 해당 요소가 차지하던 너비와 높이 공간이 완전히 소멸된다. 마치 HTML 코드에서 삭제한 것처럼, 뒤에 있던 요소들이 그 빈자리를 채우며 당겨져 온다.
3. 이벤트 차단: 클릭, 호버 등 모든 마우스 이벤트가 작동하지 않는다.
4. 자식 요소 포함: 부모에게 display: none을 주면 그 안에 있는 자식 요소들도 모두 함께 사라진다.
5. 접근성 (Screen Reader): 시각 장애인을 위한 스크린 리더도 이 요소를 읽지 않고 무시한다.

# 성능관점
display: none ↔ block 변경 시
* 요소가 없어지거나 생기면서 주변 요소들의 위치가 다 바뀌어야 한다
* 브라우저는 레이아웃을 다시 계산하는 Reflow(리플로우) 과정을 거치므로 연산 비용이 비싸다.

# display: none인 요소의 하위요소가 position: fixed 또는 absolute 라면?
역시나 함께 사라진다. 브라우저가 화면을 그릴 때 **렌더링 트리(Render Tree)**라는 것을 만든다.

* DOM 트리: HTML 태그들의 족보 (부모-자식 관계)
* 렌더링 트리: 실제 화면에 그려질 요소들만 추린 것

부모에게 display: none을 주면, 브라우저는 **"이 부모와 그 하위 자식들은 렌더링 트리에 아예 포함시키지 마"**라고 판단한다. 자식이 fixed건 absolute건 상관없이, 부모라는 존재 자체가 렌더링 단계에서 배제되었기 때문에 자식도 함께 사라지는 것이다.
